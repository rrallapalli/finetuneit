# pod (torch 2.8.0+cu128, transformers 5.12.1, trl 1.7.0, unsloth 2026.6.9).

# Pinning is what stops the recurring "a newer library renamed/retightened an
# argument" breakage (evaluation_strategy, max_seq_length, tokenizer=,
# padding_free...): installs are now reproducible. To move UP a version, bump
# the pin, run `python3.12 smoke_test.py`, and confirm before a real run.
#
# torch is intentionally NOT pinned here: RunPod templates ship a CUDA-matched
# torch and the setup script reuses it. Recorded for reference: torch==2.8.0+cu128.
# unsloth is installed separately (setup script, --no-deps) and pinned there:
#   unsloth==2026.6.9, unsloth_zoo==2026.6.7
